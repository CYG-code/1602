# 美食地图打卡小程序 — 开发实施计划

> 基于 PRD 文档，按垂直切片方式拆解为可逐步实施的开发任务。
> 每个任务都是一个完整的功能路径，完成后系统处于可运行状态。

## 概述

将 PRD 中的功能拆解为 4 个阶段、16 个任务，每个任务包含明确的验收标准和验证方式。
预计工期：1人 × 14天。

## 架构决策

- 使用微信云开发（CloudBase），免服务器运维
- 地图使用小程序内置 `map` 组件（底层腾讯地图）
- 店铺数据通过云函数调用腾讯地图 WebService API 获取
- 图片存储使用云开发云存储，前端压缩后上传
- 数据库使用云开发 NoSQL 数据库

## 依赖关系图

```
环境准备（云开发 + 腾讯地图 key）
    │
    ├── 数据库集合 + 安全规则
    │       │
    │       ├── 云函数：店铺数据获取
    │       │       │
    │       │       └── 地图首页（展示店铺标记）
    │       │               │
    │       │               └── 店铺详情页
    │       │                       │
    │       │                       ├── 打卡发布
    │       │                       │       │
    │       │                       │       └── 点赞 + 评论
    │       │                       │
    │       │                       └── 导航到店
    │       │
    │       └── 用户登录
    │               │
    │               └── 个人中心（我的打卡 + 收藏）
    │
    └── TabBar + 页面骨架
```

**验收标准：**
- [ ] 云函数 `shop` 已创建并部署
- [ ] 支持 `action: "getNearby"` 参数，接收 `latitude`、`longitude`、`keyword`（可选）
- [ ] 调用腾讯地图 `/ws/place/v1/search` 接口，搜索半径 3km 内的餐饮类 POI
- [ ] 将返回的店铺数据转换为 `shops` 集合格式，使用 `sourceId` 去重后写入数据库
- [ ] 返回附近店铺列表给前端（包含 _id、name、location、category、avgRating、checkinCount）

**验证：**
- [ ] 在云开发控制台手动测试云函数，传入经纬度参数，能返回店铺列表
- [ ] 数据库 `shops` 集合中能看到导入的店铺数据
- [ ] 重复调用不会产生重复数据

**依赖：** Task 1、Task 2

**涉及文件：**

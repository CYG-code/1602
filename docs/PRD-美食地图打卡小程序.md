# 1602美食录 — 产品需求文档（PRD）

## 1. 概述

### 1.1 产品简介
一款基于微信小程序的美食地图打卡应用，用户可以在地图上浏览附近的美食店铺，查看其他人的打卡评价，并发布自己的美食打卡记录。

### 1.2 项目背景
- 团队：1602YangTeam
- 定位：先内部使用，后续开放给更多用户
- 技术栈：微信小程序 + 云开发
- 开发人员：1人
- 交付周期：1-2周（紧急）

### 1.3 目标用户
初期为小组成员及其朋友圈，后续扩展为所在城市的美食爱好者。

### 1.4 成功指标
- MVP 在 2 周内上线可用
- 地图能正确加载并显示第三方店铺数据
- 用户可以完成完整的"浏览 → 查看详情 → 发布打卡"流程

## 2. 用户故事（INVEST 格式）

### Epic 1：地图浏览

**US-1.1 查看附近美食店铺**
```
作为 一个美食爱好者
我想要 打开小程序后在地图上看到附近的美食店铺标记
以便 快速发现身边有什么好吃的

验收标准：
Given 用户授权了定位权限
When 打开小程序首页
Then 地图以当前位置为中心显示
And 附近的美食店铺以标记点形式展示在地图上
And 标记点可点击，弹出店铺简要信息卡片（店名、评分、距离）
```

**US-1.2 筛选店铺**
```
作为 一个有明确需求的用户
我想要 按菜系、价格、评分、距离筛选地图上的店铺
以便 快速找到符合我当前需求的餐厅

验收标准：
Given 地图上已加载店铺标记
When 用户点击筛选按钮
Then 显示筛选面板，包含：
  - 菜系分类（川菜、火锅、日料、西餐、烧烤、小吃等）
  - 价格区间（0-30、30-60、60-100、100+）
  - 排序方式（评分最高、距离最近）
And 选择筛选条件后地图标记实时更新
```

**US-1.3 搜索店铺**
```
作为 一个知道想吃什么的用户
我想要 通过搜索栏输入店名或菜系关键词
以便 直接找到目标店铺

验收标准：
Given 用户在首页顶部搜索栏输入关键词
When 输入完成并触发搜索
Then 显示匹配的店铺列表
And 点击某个结果后地图定位到该店铺并弹出信息卡片
```

### Epic 2：店铺详情

**US-2.1 查看店铺详情**
```
作为 一个对某家店感兴趣的用户
我想要 点击店铺查看详细信息和其他人的打卡评价
以便 决定要不要去这家店

验收标准：
Given 用户点击地图标记或搜索结果中的店铺
When 进入店铺详情页
Then 显示以下信息：
  - 店铺名称、地址、电话、菜系标签
  - 综合评分（所有打卡评分的均值）
  - 人均消费
  - 打卡评价列表（按时间倒序）
And 每条打卡显示：用户头像昵称、图片、文字、评分、人均消费、点赞数、时间
```

**US-2.2 导航到店**
```
作为 一个决定去某家店的用户
我想要 一键导航到该店铺
以便 快速找到路线

验收标准：
Given 用户在店铺详情页
When 点击"导航"按钮
Then 调用微信内置地图导航，打开路线规划
```

### Epic 3：打卡发布

**US-3.1 发布美食打卡**
```
作为 一个刚吃完好吃的用户
我想要 发布一条打卡记录分享我的体验
以便 记录美食经历并帮助其他人做选择

验收标准：
Given 用户已登录
When 在店铺详情页点击"我要打卡"
Then 进入打卡发布页，包含：
  - 图片上传（最多9张，支持拍照和相册选择）
  - 文字评价输入框
  - 星级评分（1-5星，必填）
  - 人均消费输入（数字，选填）
And 提交后打卡出现在该店铺的评价列表中
And 店铺的综合评分和打卡数自动更新
```

### Epic 4：社交互动

**US-4.1 点赞打卡**
```
作为 一个浏览打卡的用户
我想要 给觉得有用的打卡点赞
以便 表达认同并帮助优质内容被更多人看到

验收标准：
Given 用户已登录，正在查看打卡列表
When 点击某条打卡的点赞按钮
Then 点赞数 +1，按钮变为已点赞状态
And 再次点击取消点赞，点赞数 -1
And 同一用户对同一条打卡只能点赞一次
```

**US-4.2 评论回复**
```
作为 一个想交流的用户
我想要 在打卡下方发表评论
以便 和打卡作者或其他用户交流

验收标准：
Given 用户已登录，正在查看某条打卡
When 点击评论按钮并输入评论内容
Then 评论显示在该打卡下方
And 支持回复某条评论（@某人）
And 评论按时间正序排列
```

**US-4.3 微信分享**
```
作为 一个发现好店的用户
我想要 把店铺或打卡分享给微信好友/群
以便 推荐给朋友

验收标准：
Given 用户在店铺详情页或某条打卡
When 点击分享按钮或使用微信右上角分享
Then 生成分享卡片（包含店名、评分、封面图）
And 好友点击卡片可直接进入该店铺详情页
```

### Epic 5：个人中心

**US-5.1 微信登录**
```
作为 一个新用户
我想要 用微信快捷登录
以便 无需注册即可使用

验收标准：
Given 用户未登录
When 尝试发布打卡/点赞/评论时
Then 弹出登录提示，引导微信授权
And 授权后获取用户头像和昵称
And 浏览地图和店铺详情不需要登录
```

**US-5.2 我的打卡**
```
作为 一个活跃用户
我想要 查看自己发布过的所有打卡记录
以便 回顾我的美食足迹

验收标准：
Given 用户已登录，进入个人中心
When 点击"我的打卡"
Then 按时间倒序显示我发布的所有打卡
And 每条显示店铺名、图片缩略图、评分、时间
```

**US-5.3 收藏店铺**
```
作为 一个种草了某家店的用户
我想要 收藏这家店
以便 以后想去的时候快速找到

验收标准：
Given 用户已登录
When 在店铺详情页点击收藏按钮
Then 店铺加入"我的收藏"列表
And 收藏按钮变为已收藏状态
And 在个人中心可查看所有收藏的店铺
```

## 4. 数据模型

### 4.1 云数据库集合

```
集合: shops（店铺）
├── _id              // 自动生成
├── name             // 店铺名称 string
├── address          // 详细地址 string
├── location         // GeoPoint { type: "Point", coordinates: [lng, lat] }
├── phone            // 联系电话 string
├── category         // 菜系分类 string（川菜/火锅/日料/西餐/烧烤/小吃/其他）
├── coverImage       // 封面图 fileID string
├── avgRating        // 平均评分 number（云函数计算更新）
├── avgPrice         // 人均消费 number（云函数计算更新）
├── checkinCount     // 打卡数 number（云函数计算更新）
├── source           // 数据来源 string（"tencent_map" / "user"）
├── sourceId         // 第三方POI ID string
├── createdAt        // 创建时间 date

集合: checkins（打卡）
├── _id              // 自动生成
├── shopId           // 关联店铺ID string
├── shopName         // 冗余店铺名（列表展示用）string
├── userId           // 用户openid string
├── userInfo         // { nickName, avatarUrl } object
├── images           // 图片fileID数组 array（最多9张）
├── content          // 文字评价 string
├── rating           // 评分 number（1-5）
├── avgPrice         // 本次人均消费 number
├── likeCount        // 点赞数 number
├── commentCount     // 评论数 number
├── createdAt        // 打卡时间 date

集合: comments（评论）
├── _id              // 自动生成
├── checkinId        // 关联打卡ID string
├── userId           // 评论者openid string
├── userInfo         // { nickName, avatarUrl } object
├── content          // 评论内容 string
├── replyTo          // 回复目标 { userId, nickName } object（可选）
├── createdAt        // 评论时间 date

集合: likes（点赞，防重复）
├── _id              // 自动生成
├── checkinId        // 关联打卡ID string
├── userId           // 点赞者openid string
├── createdAt        // 点赞时间 date

集合: users（用户）
├── _id              // 自动生成
├── openid           // 微信openid string
├── nickName         // 昵称 string
├── avatarUrl        // 头像URL string
├── checkinCount     // 打卡总数 number
├── favoriteShops    // 收藏的店铺ID数组 array
├── createdAt        // 注册时间 date
├── updatedAt        // 更新时间 date
```

### 4.2 数据库索引建议
- shops: `location`（2dsphere 地理索引）、`category`、`avgRating`
- checkins: `shopId + createdAt`（复合索引）、`userId + createdAt`
- likes: `checkinId + userId`（唯一复合索引，防重复点赞）
- comments: `checkinId + createdAt`

## 5. 技术方案

### 5.1 第三方数据源：腾讯地图 WebService API

推荐理由：
- 微信小程序内置 `map` 组件底层就是腾讯地图，兼容性最好
- 腾讯位置服务有免费额度（日调用 10000 次），足够初期使用
- 提供"周边搜索"接口，可按关键词搜索餐饮类 POI

关键接口：
- `周边搜索 /ws/place/v1/search`：按经纬度搜索附近餐饮店铺
- `关键词搜索 /ws/place/v1/suggestion`：输入联想补全

使用方式：
- 通过云函数调用腾讯地图 API（避免在前端暴露 key）
- 首次加载某区域时拉取 POI 数据缓存到 shops 集合
- 用户打卡时如果店铺不在库中，自动从搜索结果导入

### 5.2 页面结构
```
miniprogram/
├── pages/
│   ├── index/           → 地图首页（TabBar页）
│   ├── shopDetail/      → 店铺详情

│   ├── profile/         → 个人中心（TabBar页）
│   ├── myCheckins/      → 我的打卡列表
│   └── myFavorites/     → 我的收藏列表

```

### 5.3 云函数结构
```

```

## 8. 风险与注意事项

1. **腾讯地图 API 配额**：免费额度日调用 10000 次，初期足够，需在腾讯位置服务申请 key
2. **小程序审核**：如果要正式发布，需要准备好相关资质（个人主体可能受限）
3. **图片存储成本**：云开发免费额度 5GB 存储，打卡图片需做压缩（建议前端压缩到 500KB 以内）
4. **定位权限**：用户拒绝定位时需要有兜底方案（手动选择城市或搜索）
5. **数据安全**：云数据库权限规则需配置好，防止用户篡改他人数据
